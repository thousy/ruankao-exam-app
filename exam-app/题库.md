# 软考题库 Excel 模板编写指南 📝

本指南旨在详细介绍**软考刷题系统 V2.0**中 `questions_bank.xlsx` 题库的结构和编写规范，帮助您轻松管理、录入和扩展题库，特别是**单选题**与**多项选择题（双空/多空题）**的录入规范。

---

## 一、 题库结构总览

题库由一个 Excel 文件（`questions_bank.xlsx`）组成，内部包含两种类型的工作表（Sheet）：
1. **`catalog` 工作表**：章节目录注册表（有且仅有一个）。
2. **章节题目工作表**：以章节 ID 命名的工作表（如 `ch4`, `ch6`, `ch27` 等，一章对应一个 Sheet）。

### 1. `catalog` 工作表结构
目录表用来注册系统可以显示和练习的章节。**只有在此表中注册的章节，系统才允许导入对应的题目分表。**

| 列名 (Header) | 说明 | 示例 |
| :--- | :--- | :--- |
| **`tags`** | 章节 ID 标识（必须与题目分表名完全一致） | `ch6` |
| **`tag_contents`** | 章节的中文名称，会在系统“章节练习”主页显示 | `第6章 风险与沟通管理` |
| **`num`** | 本章包含的题目总数（展示在主页进度条中） | `134` |

> [!IMPORTANT]
> **主拦截过滤机制**：如果在导入的 Excel 中，某章节（例如 `ch4`）在 `catalog` 表中不存在，即使该 Excel 中含有 `ch4` 的题目分表，系统也会**拦截并跳过**该章节的所有题目，不会将其导入题库。

## 二、 题目分表字段规范

每一章的题目分表（如 `ch6`）包含了该章所有的题目，每一行代表一道题。共有 14 列（**本系统全新升级，已彻底消灭晦涩的 JSON 字段列，多项选择题完全通过普通的行级平铺进行极简录入**）：

| 列序号 | 列名 (Header) | 类型 | 必填 | 说明 |
| :---: | :--- | :---: | :---: | :--- |
| 1 | **`id`** | 文本 | **是** | 题目的唯一标识 ID。<br>• 普通单选题：`章节ID_序号`，如 `ch6_3`。<br>• 多空选择题：`章节ID_序号_空序号`，如第一个空为 `ch6_25_1`，第二个空为 `ch6_25_2`。 |
| 2 | **`chapterId`** | 文本 | **是** | 所属章节 ID，必须与分表名一致，如 `ch6`。 |
| 3 | **`type`** | 文本 | **是** | 题目类型。单选题填 **`single`**；多空选择题填 **`multi`**。 |
| 4 | **`difficulty`** | 文本 | 否 | 难度级别。可选值：`easy`（简单）、`medium`（中等，默认）、`hard`（困难）。 |
| 5 | **`content`** | 文本 | **是** | 题目的题干内容。支持 HTML 标签（如换行 `<br>` 或图片引用）。**多空题的每一行都要填入相同的公共大题干。** |
| 6 | **`option_A`** | 文本 | 否 | 选项 A 的内容。 |
| 7 | **`option_B`** | 文本 | 否 | 选项 B 的内容。 |
| 8 | **`option_C`** | 文本 | 否 | 选项 C 的内容。 |
| 9 | **`option_D`** | 文本 | 否 | 选项 D 的内容。 |
| 10 | **`correctAnswer`**| 数值 | 否 | 正确答案索引。填 **`0`** (代表A), **`1`** (B), **`2`** (C), **`3`** (D)。 |
| 11 | **`explanation`** | 文本 | 否 | 题目的答案解析。支持换行及详细的推导公式。 |
| 12 | **`tags`** | 文本 | 否 | 题目标签，多个标签用英文逗号 `,` 分隔。通常填章节 ID。 |
| 13 | **`image`** | 文本 | 否 | 题干关联的图片文件名。图片统一存放在项目的 `images/` 目录下（如 `ch6_q1.png`）。 |
| 14 | **`explanationImage`**| 文本 | 否 | 解析关联的图片文件名，统一放在 `images/` 目录下。 |

## 三、 单选题编写规则 (`single`)

单选题是系统中最基础的题目类型。
* **`type`**：填 `single`
* **`id`**：格式如 `ch6_3`
* **`option_A` ~ `option_D`**：分别填入四个选项的文本。
* **`correctAnswer`**：对应正确答案的数字序号：
  * **`0`** 对应 A
  * **`1`** 对应 B
  * **`2`** 对应 C
  * **`3`** 对应 D

### Excel 录入示例：
| id | chapterId | type | content | option_A | option_B | option_C | option_D | correctAnswer | explanation |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :---: | :--- |
| `ch6_3` | `ch6` | `single` | 该项目的期望货币价值为（ 69 ）美元。 | `20000` | `85000` | `50000` | `180000` | `1` | EMV=0.5*200000 - ... |

## 四、 多项选择题 / 双空选择题编写规则 (`multi`)

在软考中，经常会出现一道大题包含两个或三个选择空格的情况（例如：一问最快完成时间是多少，二问通过什么活动可以缩短工期）。系统为此设计了 **`multi`**（多空选择）模式，可以在同一道题下生成多个独立的答题卡。

### 核心设计：平铺多行 + ID 命名规则
**多空题不需要在同一行内塞入复杂的 JSON 语法，也无需任何 SLOT 字段。您只需要为每个空分别添加一行数据！**

#### 1. 规则要点：
* **`type`**：全部填 **`multi`**。
* **`id` 字段（识别合并的关键）**：
  * 第一行的 ID 命名为 `大题ID_1`（如 `ch6_25_1`），代表**第 1 个空**的题目、选项与答案。
  * 第二行的 ID 命名为 `大题ID_2`（如 `ch6_25_2`），代表**第 2 个空**的题目、选项与答案。
  * 第三行的 ID 命名为 `大题ID_3`（如 `ch6_25_3`），代表**第 3 个空**，依此类推。
* **`content` (大题干)**：多空题的所有行必须填入**完全相同的公共大题干内容**。
* **选项及答案**：
  * 每一行分别填入该空对应的 `option_A` ~ `option_D`。
  * 每一行分别填入该空对应的 `correctAnswer`（0 代表 A，1 代表 B，2 代表 C，3 代表 D）。
* **公共属性**：像解析（`explanation`）、图片（`image`、`explanationImage`）、标签（`tags`）等大题属性，直接在多行的第一行（或每一行）填入即可，系统会自动合并并以第一行的内容兜底展示。

---

### 2. 双空选择题录入 Excel 实操演示：

假设有一道风险计算大题：
> 某工程建设项目中各工序历时如下表所示，则本项目最快完成时间为 **（ 35 ）** 周。同时，通过 **（ 36 ）** 可以缩短项目工期。
> 第一空选项：A. 7  B. 9  C. 12  D. 13 （答案是 D）
> 第二空选项：A. ①⑤  B. ①③  C. ⑤  D. ③④ （答案是 A）

在 Excel 中只需占用**两行**这样轻松填写：

| id (ID) | type (类型) | content (公共大题干) | option_A | option_B | option_C | option_D | correctAnswer (答案) | explanation (解析) |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :---: | :--- |
| **`ch6_25_1`** | **`multi`** | 某工程建设项目中...本项目最快完成时间为 （ 35 ） 周。同时... | `7` | `9` | `12` | `13` | **`3`** (代表D) | 本题考查关键路径计算。关键路径总工期为 13 周... |
| **`ch6_25_2`** | **`multi`** | 某工程建设项目中...本项目最快完成时间为 （ 35 ） 周。同时... | `①⑤` | `①③` | `⑤` | `③④` | **`0`** (代表A) | 本题考查关键路径计算。关键路径总工期为 13 周... |

### 3. 系统后台智能合并原理：
当用户在网页端导入该 Excel 时，系统会自动检测 `type` 为 `multi` 且带有下划线数字后缀（如 `_1`、`_2`）的行。系统会在内存中自动将 `ch6_25_1` 和 `ch6_25_2` 合并为一道拥有两个空的大题 `ch6_25`，并自动构建完美贴合的前端“双空/多空答题卡”！
* 练习时，本题只算作 **1 道题**（而不是 2 道）。
* 答题卡会清晰展示为：**【考点 1】** 的选项卡 与 **【考点 2】** 的选项卡。
* 用户可以分别切换、作答、校验，完美契合软考真题的考试体验！

## 五、 带图题目编写规则

如果题目中包含图表（网络图、决策树、收益表等）：
1. 把图片切图后，统一保存为 PNG 或 JPG 格式（推荐命名如 `ch6_q25.png`）。
2. 将图片放置到项目目录下的 **`images/`** 文件夹中。
3. 在 Excel 编写时，有两种方式引用图片：
   * **推荐方式（直接在表格列中配置）**：
     在 **`image`** 列填写图片文件名（如 `ch6_q25.png`），在 **`explanationImage`** 列填写解析图片文件名。系统会自动在题干下方或解析下方渲染它。
   * **HTML 嵌入方式**：
     在 `content` 文本的任意位置插入 `<img src="images/ch6_q25.png" class="question-img">` 标签。这允许您自由控制图片在文字中间的相对排版位置。

## 六 & 编写注意事项与防错贴士 💡

1. **大题 ID 一致性**：同一道多空题的所有行，下划线前面的主 ID 部分必须**大小写和拼写完全一致**。例如必须是 `ch6_25_1` 和 `ch6_25_2`，如果一处写了 `ch6_25_1` 另一处写了 `CH6_25_2`，系统会因大小写不一致而无法合并。
2. **大题干一致性**：多选题各行的公共大题干 `content` 最好完全相同。
3. **章节标签对齐**：题目分表的名字（如 `ch6`）、`catalog` 表中的 `tags` 字段（如 `ch6`）、以及题目行中的 `chapterId` 字段（如 `ch6`）三者必须**大小写和拼写完全一致**，不能有空格。
