## 📂 题库基本大纲结构

题库由一个 Excel 文件（`questions_bank.xlsx`）组成，内部包含两种类型的工作表（Sheet）：
1. **`catalog` 工作表**：章节目录注册表（有且仅有一个）。
2. **章节题目工作表**：以章节 ID 命名的工作表（如 `ch4`, `ch6`, `ch27` 等，一章对应一个 Sheet）。

### 1. `catalog` 工作表注册
只有在此表中注册的章节，系统才允许导入对应的分表。

| 列名 (Header) | 说明 | 示例 |
| :--- | :--- | :--- |
| **`tags`** | 章节 ID 标识（必须与题目分表名完全一致） | `ch6` |
| **`tag_contents`** | 章节的完整中文名称，会在系统主页显示 | `第六章 计算专项` |
| **`num`** | 本章包含的题目总数（展示在主页进度条中） | `134` |

### 2. 章节分表 15 列标准字段
本系统全新升级，已彻底消灭晦涩的 JSON 列，支持通过以下 15 列直接进行平铺录入：
1. **`id`** (唯一标识)
2. **`chapterId`** (所属章节)
3. **`type`** (题目类型：`single` 代表单选，`multi` 代表多项/多空选择题)
4. **`difficulty`** (难度：`easy`, `medium`, `hard`)
5. **`label`** (考点卡片名称，如 `"请选择答案"`、`"考点 66"`。**最新升级 🌟**)
6. **`content`** (公共题干，多选题各平铺行填入完全相同的内容)
7. **`option_A`** (选项 A)
8. **`option_B`** (选项 B)
9. **`option_C`** (选项 C)
10. **`option_D`** (选项 D)
11. **`correctAnswer`** (正确答案：`0` 对应 A，`1` 对应 B，`2` 对应 C，`3` 对应 D)
12. **`explanation`** (解析内容)
13. **`tags`** (标签，用英文逗号分隔)
14. **`image`** (题干图片，存放在项目同级 `images/` 中)
15. **`explanationImage`** (解析图片，存放在 `images/` 中)

## 🚀 核心编写规则速查
* **单选题**：`type` 填 `single`，ID 命名如 `ch6_3`。
* **平铺多选题（一题多问）**：`type` 填 `multi`。第一空 ID 填 `章节ID_序号_1`（如 `ch6_25_1`），第二空 ID 填 `章节ID_序号_2`（如 `ch6_25_2`），公共大题干内容填入完全相同的一致文本。
* **图片文件**：在 `image` 或 `explanationImage` 列填写图片的文件名（例如 `ch6_q25.png`），切图并统一放置在 `images/` 目录下。
