# 智能做题系统 (原名：软考项目管理做题应用) 💻

## 📚 项目简介

这是一个专为软考（软件水平考试）项目管理科目设计的现代化在线做题应用。采用**纯前端无服务器技术栈**，CORS 免疫，所有数据存储在浏览器本地，完全离线运行。您只需双击 `index.html` 即可在几毫秒内闪电启动，开启您的离线极速复习之旅！

---

## ✨ 核心功能

### 🎯 学习与答题
* **章节练习**：按照教材章节系统化学习，循序渐进，显示各章节的进度与正确率。
* **随机练习**：随机抽取题目检验知识，特别适合考前冲刺检验。
* **错题本**：答错的题会自动收集，重新答对后自动移出，方便查漏补缺。
* **收藏夹**：点击题目星标图标随时收藏，供考前反复记忆。
* **搜索功能**：在题库中快速查找特定关键字或特定章节的题目。

### 🎨 高端用户体验与交互
* **深色/浅色护眼主题**：一键切换夜间深色和白天浅色主题，保护视力。
* **响应式流动布局**：完美自适应适配桌面、平板和所有智能手机。
* **图片高级缩放与查看**：
  - 点击题目或解析中的图片，可打开模态框全屏查看。
  - **滚轮缩放**：滚轮向上滚动放大（最大5倍），向下滚动缩小（最小0.5倍）。
  - **鼠标拖动**：放大后支持鼠标拖拽，方便查看复杂的网络图与收益矩阵细节。
  - **双击重置**：双击图片恢复到原始大小和位置。
* **多空选择题（一题多问）**：
  - 自动识别多空格题目，为每个空格生成独立的答题卡。
  - 支持分别作答、独立校验，并支持针对各考点分别显示解析。

## 🚀 极速启动与运行

### 方法一：双击直接打开 (推荐)
1. 进入应用目录。
2. 双击 **`index.html`** 文件。
3. 浏览器将瞬间离线开启系统，无需配置任何网络代理。

### 方法二：启动轻量本地服务器 (可选)
如果您想通过局域网在手机端测试，可以快速启动 Python 内置服务器：
```bash
# 进入项目目录运行
python -m http.server 8000
```
然后用浏览器访问：👉 **`http://localhost:8000`**

## 📝 题库 Excel (questions_bank.xlsx) 编写与导入指南

本系统已彻底消灭晦涩难懂的 JSON 列，支持使用 Excel (**`questions_bank.xlsx`**) 进行题库批量离线录入。系统内部采用自然物理顺序装配引擎，保证题目在导入后的物理行顺序与 Excel 中一模一样。

### 1. 题库表结构总览
Excel 文件由两类工作表（Sheet）组成：
1. **`catalog` 工作表**：章节目录注册表（有且仅有一个）。
2. **章节题目工作表**：以章节 ID 命名的题目分表（如 `ch4`, `ch6`, `ch27` 等，一章对应一个 Sheet）。

#### 📌 `catalog` 工作表规范
只有在目录表中注册的章节标识，系统在导入时才会允许对应的分表导入，否则会拦截并跳过。

| 列名 (Header) | 说明 | 示例 |
| :--- | :--- | :--- |
| **`tags`** | 章节 ID 标识（必须与分表名完全一致） | `ch6` |
| **`tag_contents`** | 章节的完整中文名称，会在系统“章节练习”主页显示 | `第六章 计算专项` |
| **`num`** | 本章包含的题目总数（展示在主页进度条中） | `134` |

---

### 2. 章节分表字段定义规范
每一章的分表（如 `ch6`）包含了该章所有的题目，每一行代表一道题目。Excel 文件包含以下 15 列：

| 列序号 | 列名 (Header) | 类型 | 必填 | 说明 |
| :---: | :--- | :---: | :---: | :--- |
| 1 | **`id`** | 文本 | **是** | 题目的唯一标识 ID。<br>• 普通单选题：`章节ID_序号`，如 `ch6_3`。<br>• 平铺多选题：`章节ID_大题号_空序号`，如第一个空为 `ch6_25_1`，第二个空为 `ch6_25_2`。 |
| 2 | **`chapterId`** | 文本 | **是** | 所属章节 ID，必须与分表名一致，如 `ch6`。 |
| 3 | **`type`** | 文本 | **是** | 题目类型。单选题填 **`single`**；多选/多空题填 **`multi`**。 |
| 4 | **`difficulty`** | 文本 | 否 | 难度级别。可选值：`easy`（简单）、`medium`（中等，默认）、`hard`（困难）。 |
| 5 | **`label`** | 文本 | 否 | **自定义考点标签列（全新升级 🌟）**。例如平铺多选题的各空可填入如 `"请选择答案"`、`"考点 66"`、`"核心考点"`等，系统导入时会自动渲染在答题卡的卡片名称上。如果留空，系统会自动优雅兜底显示为 `"考点 {序号}"`。 |
| 6 | **`content`** | 文本 | **是** | 题目的题干内容。支持 HTML 标签（如换行 `<br>`）。**多选题的各个平铺行必须填入完全相同的公共大题干。** |
| 7 | **`option_A`** | 文本 | 否 | 选项 A 的内容。 |
| 8 | **`option_B`** | 文本 | 否 | 选项 B 的内容。 |
| 9 | **`option_C`** | 文本 | 否 | 选项 C 的内容。 |
| 10 | **`option_D`** | 文本 | 否 | 选项 D 的内容。 |
| 11 | **`correctAnswer`**| 数值 | 否 | 正确答案索引。填 **`0`** (代表A), **`1`** (B), **`2`** (C), **`3`** (D)。 |
| 12 | **`explanation`** | 文本 | 否 | 题目的答案解析。支持换行及详细的推导公式。 |
| 13 | **`tags`** | 文本 | 否 | 题目标签，多个标签用英文逗号 `,` 分隔。通常填章节 ID。 |
| 14 | **`image`** | 文本 | 否 | 题干关联的图片文件名。图片统一存放在项目同级的 `images/` 目录下（如 `ch6_q25.png`）。系统会自动在题干下方渲染它。 |
| 15 | **`explanationImage`**| 文本 | 否 | 解析关联的图片文件名，统一放在 `images/` 目录下。 |

---

### 3. 单选题录入示例 (`single`)
* **`type`** 填 `single`
* **`id`** 填如 `ch6_3`
* **`correctAnswer`** 填对应答案索引（0 代表 A，1 代表 B，2 代表 C，3 代表 D）。

| id | chapterId | type | content | option_A | option_B | option_C | option_D | correctAnswer | explanation |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :---: | :--- |
| `ch6_3` | `ch6` | `single` | 该项目的期望货币价值为（ 69 ）美元。 | `20000` | `85000` | `50000` | `180000` | `1` | EMV = 0.5 * 200000 - ... |

---

### 4. 多项选择题 / 双空选择题平铺录入规范 (`multi`)
在软考中，经常会出现一道大题包含两个或三个连续空格的情况。系统对此设计了 **`multi`**（多空选择）模式，**不需要在同一行内编写复杂的 JSON 嵌套。您只需要将每个空平铺成一行即可！**

#### 📌 平铺录入规则：
1. **`type`**：所有平铺行必须填 **`multi`**。
2. **`id` 字段（核心合并标识）**：
   - 第一行的 ID 命名为 `大题ID_1`（如 `ch6_25_1`），代表第 1 个空。
   - 第二行的 ID 命名为 `大题ID_2`（如 `ch6_25_2`），代表第 2 个空，依此类推。
3. **`label` 字段**：输入该行对应的小题名字（如 `“请选择答案”`，`“考点 66”` 等）。
4. **`content` (公共大题干)**：平铺的所有行必须填入**完全一致的公共大题干内容**。
5. **选项与答案**：每一行分别独立填入各自的 `option_A` ~ `option_D` 与 `correctAnswer` 索引。
6. **公共属性**：像解析（`explanation`）、图片（`image`、`explanationImage`）等大题属性，直接在首行填入即可，系统会自动合并并进行优雅的兜底渲染。

#### Excel 录入实操演示：
| id | type | label | content | option_A | option_B | option_C | option_D | correctAnswer |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- | :---: |
| **`ch6_25_1`** | **`multi`** | `请选择答案` | 某工程建设项目中...本项目最快完成时间为 （ 35 ） 周。同时... | `7` | `9` | `12` | `13` | **`3`** (D) |
| **`ch6_25_2`** | **`multi`** | `考点 66` | 某工程建设项目中...本项目最快完成时间为 （ 35 ） 周。同时... | `①⑤` | `①③` | `⑤` | `③④` | **`0`** (A) |

#### 后台智能合并原理：
当您在网页端选择 `📤 导入 Excel 题库` 并选择该文件时，系统会在浏览器内存中自动将 `ch6_25_1` 和 `ch6_25_2` 物理行融合成一道包含 2 个空格的大题 `ch6_25`。
在练习界面中，这只算作 **1 道题**（而不是 2 道），并且答题卡会生成两个独立分页卡片：**【请选择答案】** 与 **【考点 66】**，提供极其 Premium、专业的答题体验！

---

### 5. 带图题目与解析图片引用方法
* **直接列配置（推荐）**：在 Excel 的 `image` 或 `explanationImage` 中填入文件名（如 `ch6_q25.png`），并将对应的 PNG/JPG 文件放入项目目录同级的 **`images/`** 目录中即可。
* **HTML 自由嵌入**：在 `content` 或 `explanation` 文本的任意位置，可以使用 `<img src="images/ch6_q25.png" class="question-img">` 实现文本混排。

---

### ⚠️ 编写注意事项与防错贴士
1. **主 ID 拼写完全一致**：同一道多选题的各平铺行，下划线前的父 ID 部分必须大小写和拼写完全相同（例如 `ch6_25_1` 和 `ch6_25_2` 的父 ID `ch6_25` 必须完全匹配）。
2. **三方对齐**：章节 ID（如 `ch6`）、分表名（如 `ch6`）、以及 `catalog` 表中的 `tags` 必须完全对齐，不能含有空格。

## 💾 数据备份与清除

### 1. 数据本地导出
打开浏览器开发者工具控制台（F12），执行以下代码将您的本地学习进度、收藏夹和错题本打包为 JSON 字符串复制出来备份：
```javascript
const backupData = {
    progress: localStorage.getItem('exam_user_progress'),
    answers: localStorage.getItem('exam_user_answers'),
    favorites: localStorage.getItem('exam_favorites'),
    mistakes: localStorage.getItem('exam_mistakes'),
    stats: localStorage.getItem('exam_study_stats')
};
console.log(JSON.stringify(backupData, null, 2));
```

### 2. 重置清除所有数据
如需清空所有历史做题进度，直接在浏览器控制台执行：
```javascript
localStorage.clear();
location.reload();
```
